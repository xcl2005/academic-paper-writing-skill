# 14 Undergraduate Thesis and Graduation Project Engine

## Purpose

Support ordinary undergraduate thesis / graduation design without falsely applying top-tier paper standards.

## Key Principle

Graduation design evaluation often includes implicit judgments of effort, completeness, independence, and presentation, but these are not universal numeric standards. Do not output fake exact workload scores.

## Workflow

1. Discover official and advisor-specific requirements.
2. Define project scope using Scope Ladder.
3. Build MVP first.
4. Add evidence-producing modules.
5. Create implementation/test/evidence logs.
6. Write thesis around what was actually built and verified.
7. Prepare defense with likely Q&A.

## Common Thesis Structure

- Introduction / background.
- Requirement analysis.
- Related technology / literature review.
- System or method design.
- Implementation.
- Testing and evaluation.
- Results and discussion.
- Conclusion and future work.

## Evidence Map

For each contribution/module, record:

- implemented feature;
- code location;
- screenshot/log/demo;
- test case;
- thesis section;
- defense talking point;
- risk if questioned.

## Upgrade Path

If the project has research potential, upgrade only after graduation-safe evidence exists:

- add literature matrix;
- identify novelty;
- add baseline/ablation;
- refine experiments;
- rewrite as short paper or workshop paper.
