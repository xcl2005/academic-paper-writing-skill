# 04 Requirement Discovery

## Purpose

Handle unclear requirements, especially for undergraduate thesis, graduation project, and school templates.

## Principle

Do not invent requirements. When requirements are unclear, build a requirement discovery log and assumption register.

## Sources to Request or Search

- school/college thesis notice;
- official Word/LaTeX template;
- task book / proposal form / midterm form;
- defense rubric;
- advisor messages;
- previous accepted theses;
- course/project specification;
- department handbook.

## Unknown Requirement Handling

For each unknown requirement, record:

- item;
- status: confirmed / assumed / unknown / conflicts;
- source;
- impact;
- next action.

## Graduation Project Acceptance Evidence

Instead of fake numeric workload scoring, map the project to evidence:

- code modules;
- commit history;
- screenshots;
- logs;
- test cases;
- datasets;
- experiment outputs;
- deployment/demo video;
- architecture diagrams;
- thesis sections;
- defense explanation.
